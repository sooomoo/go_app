此文件夹放置的内容为项目内部使用的代码
其他项目无法通过引用pkg的方式读取到

这是go的自有规则，go 1.4+的特性，禁止被外部项目导入
使用internal目录可以有效地限制包的访问权限，因为Go编译器对此有强制性要求。
你可以在项目树的任何级别设置多个internal目录。

此处也可以不用app目录，将app目录里面的内容提升到 internal 下面
这样更扁平，看具体情况做决定

如果是微服务项目，结构可以更改为：
internal/
|----api/     # API 路由
|----domain/  # 领域模型
|----service/ # 业务逻辑层
|----storage/ # 持久化层
